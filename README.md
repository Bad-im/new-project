# FireForest Monitor

Веб-модуль предназначен для загрузки спутниковых данных, запуска процедуры классификации лесного покрова и отображения карты пожароопасности.

## Структура

```text
frontend/   React + TypeScript + Vite, карта на React-Leaflet
backend/    FastAPI API, заглушка сервиса классификации
```

## Запуск backend

```bash
cd backend
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

Проверка:

```bash
curl http://127.0.0.1:8000/health
```

## Запуск frontend

```bash
cd frontend
npm install
npm run dev
```

Frontend по умолчанию ожидает backend по адресу `http://127.0.0.1:8000`.

Если нужен другой адрес API, создайте файл `frontend/.env`:

```env
VITE_API_BASE_URL=http://127.0.0.1:8000
```

## Возможности текущего каркаса

- Единый layout с разделами: «Обзор», «Карта», «Анализ снимков», «Результаты», «О системе».
- Mock-авторизация на frontend: кнопка «Войти» включает режим администратора, «Выйти» возвращает гостевой режим. Состояние хранится в `localStorage`.
- Гость может открыть обзор, карту, фильтровать зоны пожароопасности и читать раздел «О системе».
- Администратор дополнительно может открыть «Анализ снимков», выбрать GeoTIFF, запустить тестовый анализ и просматривать таблицу результатов.
- Карта использует OpenStreetMap, тестовый GeoJSON-слой, фильтры района и класса, легенду и статистику отображения.
- `POST /predict` сохраняется и возвращает тестовый GeoJSON FeatureCollection.
- Backend-структура подготовлена для последующего подключения ML-модели.

## Разделы интерфейса

- `Обзор` - назначение системы, используемые данные, доступ пользователей и статистика-заглушка.
- `Карта` - интерактивная карта с фильтрами, слоями, popup по зонам и легендой классов.
- `Анализ снимков` - защищенный раздел администратора для выбора GeoTIFF и запуска тестового анализа.
- `Результаты` - защищенный раздел администратора с тестовой таблицей обработок.
- `О системе` - краткое техническое описание прототипа, ролей, входных данных и ограничений.

## Роли

Гость:

- открывает сайт;
- просматривает обзор и карту;
- использует фильтры карты;
- просматривает зоны пожароопасности и popup-информацию;
- открывает раздел «О системе».

Администратор:

- имеет все возможности гостя;
- загружает GeoTIFF в тестовом интерфейсе;
- запускает тестовый анализ через `POST /predict`;
- открывает таблицу результатов обработки.
